Appendix C Linker Script Commands

The following is an alphabetical listing of the script commands recognized by the linker and the arguments they may take. Note that not all arguments are mandatory and some are mutually exclusive.

bank { size=size, origin=origin-address }
Start a new bank of size size bytes and set initial relocation address to origin-address.
copy { file=filename }
Copy contents of filename to output.
link { file=filename, origin=origin-address }
Relocate code in the unit filename to origin-address and copy the result to output. (If an origin is not specified, the internally managed linker origin is used.)
output { file=filename }
Set the linker output file.
pad { origin=origin-address, offset=offset, size=size }
Pad to the given origin or bank offset, or pad size bytes (only one of the arguments should be given).
ram { start=start-address, end=end-address }
Specify that 6502 RAM in the range start-address...end-address (non-inclusive) may be used by the linker to map contents of data segments.
