Appendix F Error and Warning Messages

F.1 Assembler Error Messages

cannot expand `identifier'; not a macro
Make sure identifier is a macro and not a label, constant or other type of symbol.
conditional expression does not evaluate to literal
Conditional assembly with the .if-directive requires that the expression tested can be evaluated immediately, so it can't contain references to labels and such (since these aren't computed by the assembler, that's the linker's job).
could not open `filename' for reading
Check that the file exists and that you have read privileges.
duplicate symbol `identifier'
You tried to define the same symbol more than once. Global labels must be unique across the entire program. Local labels must be unique within the relevant local scope.
field declaration expected
A structure or union field must be of the form identifier datatype [count]; for example, my_field .db, my_2nd_field .dsw 10, my_3rd_field .type other_struc.
data initialization not allowed here
A structure or union declaration cannot contain initialization of its fields.
initializer does not evaluate to integer literal
A member of an enumerated datatype must be assigned a constant value.
initializer for field `identifier' exceeds field size
When defining the value of a structure or union field, the value must not exceed the number of bytes of storage allocated for that field. For example, if the field definition is a_string .dsb 4, then the value "too long" won't fit since it is 8 bytes long.
instructions not allowed in data segment
Instructions can only be contained in a code segment (after a codeseg directive).
invalid addressing mode
The combination of mnemonic and addressing mode for the 6502 instruction is invalid. For example, LDX $00,X is invalid since the LDX instruction does not have a ZeroPage,X-mode version. Consult a 6502 manual to see what modes are valid for each instruction.
invalid dataseg statement
A data segment only supports a subset of the statements allowed in code segments. You can't put instructions or initialized data in a data segment.
invalid operand
You supplied an invalid operand to a statement, for example a string as operand to the LDA instruction.
macro `identifier' does not take count argument(s)
You supplied the wrong amount of arguments to the macro. Check the macro definition if you're unsure how many arguments it takes, and try again.
member `sub-struct-identifier' of `struct-identifier' is not a structure
The expression struct-identifier.sub-struct-identifier.some-member did not resolve because sub-struct-identifier is not a structure.
member 'field-identifier' of 'struct-identifier' is of unknown type (`type-identifier')
type-identifier is an undefined type.
only one field of union can be initialized
When defining an instance of a union, only one of the possible fields can be given a value between the pair of enclosing braces { }.
operand out of range
A 6502 instruction has either an 8-bit or 16-bit operand, so its value has to fit in that many bits. However, the value you supplied was too large to fit.
procedures not allowed in data segment
A procedure contains code. Code cannot be contained in a data segment.
repeat count does not evaluate to literal
Anonymous macros are expanded as soon as they are encountered. Thus, the argument of a rept directive must be an immediate expression.
size of `identifier' is unknown
The operand to the sizeof operator must be one of BYTE, WORD, DWORD, or the name of a structure definition.
string or integer argument expected
The message directive takes a string or integer as its argument.
structure initializer expected
When defining data that is an instance of a structure or union, the field value(s) must be enclosed in a pair of braces { }.
too many field initializers
There are too many values given compared to the actual number of fields in the structure or union.
union member must be of constant size
The size of a union member must be known at assembly time. This restriction does not apply to structs.
unknown macro or directive `identifier'
You attempted to invoke a macro or directive that the assembler doesn't recognize. Check your spelling (remember that identifiers are case sensitive) and/or your macro definitions.
unknown namespace `identifier'
The expression identifier::symbol did not resolve because identifier is not a namespace.
unknown symbol `identifier'
Your code refers to a symbol which hasn't been defined locally nor has it been declared to be external.
value not allowed in data segment
Data cannot be initialized in a data segment; the data segment can only specify how many bytes of storage will be needed at runtime.
`identifier' declared as extrn but is defined locally
Defining a label in your own code and then declaring it as an external symbol doesn't make much sense.
`identifier' already declared extrn
An identifier already specified in a public directive cannot at the same time be external.
`identifier' is of non-exportable type
Macros and other volatile symbols cannot be exported.
`field-identifier' is not a member of `struct-identifier'
The expression struct-identifier.field-identifier did not resolve.

F.2 Assembler Warning Messages

`identifier' declared as public but is not defined
You cannot export a symbol that isn't defined in your code.
`identifier' defined but not used
Usually there is a reason for defining a symbol, so the assembler will warn you if there are no references to it in the code.
operand out of range; truncated
Operand exceeds 8 or 16 bits, so the upper bits are chopped off.
redefinition of `identifier' is not identical; ignored
When using the .equ-directive you can only define each identifier once. (Use the = operator instead if appropriate.)

F.3 Linker Error Messages

branch out of range
A relative branch instruction went too far. Trim your code or do an inverse-branch-followed-by-jump combo instead.
duplicate symbol `identifier'
A symbol with the same name is exported from two or more of the units being linked. When linking, exported names must be unique across all units.
incompatible operand(s) to `operator' in expression
instruction operand doesn't fit in 1 byte
A rather fatal error. A zeropage instruction's operand address won't fit.
instruction operand doesn't fit in 2 bytes
A rather fatal error which shouldn't even occur.
invalid instruction operand (string)
6502 instructions only take integer operands.
negative count
A storage directive must have a positive integer operand.
out of 6502 RAM while allocating unit `unit'
The linker couldn't map the data segments to 6502 RAM because there was too little of it available. Check your ram-commands in the script or reduce your program's memory requirements.
PC went beyond 64K when linking `unit'
unexpected string operand (`string') to storage directive
Storage directives only take integer operands.
unknown symbol `identifier' referenced from unit
The external symbol couldn't be resolved. You need to link the unit containing the symbol.

F.3.1 Linker Script Error Messages

bank size (size) exceeded by count bytes
The bank output exceeded the size of the current bank. The bank size is wrong, your code is too large or the files you are copying to the bank are.
cannot pad backwards
If you start a bank and copy a, say, 2K file to it, then attempt to pad to offset 1K you will get this error. Padding can only be done from a smaller offset to a larger or equal offset. Your pad offset is wrong or the data preceding it is too large.
could not open `filename' for reading
I'm sure you know what this means by now.
could not open `filename' for writing
The specified output file could not be created.
`end' is smaller than `start'
The end address should be larger or equal to the start address, not the other way around.
failed to load `unit'
The object file could not be loaded from storage. The file is missing, you don't have access to it or it is corrupted.
invalid size
The size must be a positive (larger than zero) quantity.
missing argument `name'
The script command requires an argument which you did not supply.
no bank size set
At a minimum, the first bank-command in the script must supply a bank size.
no output open
When executing a script command which writes to the linker's output, an output file must have been specified first. Make sure that all link-, copy-, pad-commands etc. are preceded by the proper output-command.
value of argument `name' is out of range
The script command argument's value is outside the expected range. For example, an argument which specifies a 6502 address should be between 0 and 64K.

F.4 Linker Warning Messages

`.D(B|W)' operand integer out of range; truncated
Operand exceeds 8 or 16 bits, so the upper bits are chopped off.
